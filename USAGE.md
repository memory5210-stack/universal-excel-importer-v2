# 万能导入 V2 - 使用指南

## 快速开始

### 1. 访问系统

打开部署后的 URL（例如：`https://your-app.vercel.app`）

### 2. 创建第一条解析规则

#### 方式 A：手动配置

1. 导航到"解析规则"页面
2. 点击"新建规则"按钮
3. 填写基本信息：
   - 规则名称（如：黎明屯配送发货单）
   - 文件类型（Excel/Word/PDF）
   - 描述（可选）

4. 配置规则参数：
   - **跳过的头部行数**：文件开头非表格内容的行数
   - **表头行索引**：包含列名的行（从 0 开始）
   - **数据起始行**：第一条数据的行号
   - **字段映射**：将文件列映射到系统字段

5. 点击"保存规则"

#### 方式 B：AI 辅助生成（推荐）

1. 点击"AI 生成规则"按钮
2. 上传样例文件（Excel/Word/PDF）
3. AI 会自动分析文件结构并生成推荐规则
4. 检查 AI 生成的规则，进行微调
5. 确认无误后保存

### 3. 导入文件

1. 返回首页
2. 选择或拖拽文件到上传区域
3. 选择已创建的解析规则
4. 点击"开始解析"

系统会展示：
- 上传进度条
- 解析结果预览
- 数据校验结果（错误行标红）

### 4. 预览与编辑

在预览页面，您可以：

- **查看数据**：以表格形式展示解析结果
- **编辑单元格**：点击任意单元格进行修改
- **删除行**：点击行末的"删除"按钮
- **新增空行**：点击底部"新增空行"
- **导出 Excel**：将当前数据导出为 Excel 文件

### 5. 提交下单

1. 检查数据预览，确保无错误
2. 点击"提交下单"按钮
3. 系统会进行最终校验
4. 提交成功后显示汇总：
   - 成功提交 N 条
   - 失败 0 条

### 6. 查看历史运单

导航到"已导入运单"页面：

- 查看所有历史记录
- 使用筛选/搜索：
  - 按外部编码
  - 按收件人姓名
  - 按提交时间
- 分页浏览（每页 20 条）

## 支持的复杂格式

### Excel 文件

1. **干扰头部**
   - 场景：文件开头包含公司名称、日期等非表格信息
   - 配置：设置"跳过的头部行数"

2. **跨行聚合**
   - 场景：同一外部编码下有多行 SKU，共享收货信息
   - 配置：启用"分组聚合"，设置聚合字段为"外部编码"

3. **矩阵转置**
   - 场景：SKU 为行，门店为列
   - 配置：启用"矩阵转置"，填写门店列名

4. **多 Sheet 合并**
   - 场景：一个 Excel 文件包含多个 Sheet，每个 Sheet 是独立出库单
   - 配置：启用"遍历所有 Sheet"

5. **卡片式结构**
   - 场景：非标准表格，每条记录是独立的卡片区域
   - 配置：设置"卡片边界标识"（如"▶ 调拨记录"）

6. **复合单元格拆分**
   - 场景：单个单元格内含多行文本（物品名 x 数量）
   - 配置：启用"复合单元格拆分"，设置分隔符

### Word 文件

**纯文本格式**
- 场景：无表格，段落式描述
- 配置：设置分隔符（如"━━━"）划分记录边界
- AI 会自动识别键值对格式（如："物品编码：10001"）

### PDF 文件

**多页多单**
- 场景：一个 PDF 包含多个独立订单
- 配置：设置分隔标记或页面边界
- AI 会分别解析每个订单的收货信息和物品明细

**头部元信息 + 标准表格**
- 场景：页眉包含业务信息，中间是标准表格
- 配置：设置跳过的行数，提取表格外信息

## 字段说明

### 必填字段（至少满足一组）

**A 组（门店模式）**：
- 收货门店：如"尹三顺自助烤肉（银泰店）"

**B 组（收件人模式）**：
- 收件人姓名 + 收件人电话 + 收件人地址（三个必须同时填写）

**必填**：
- SKU 物品编码
- SKU 物品名称
- SKU 发货数量（正数）

### 可选字段

- 外部编码：用于去重和聚合
- SKU 规格型号
- 备注

## 错误处理

### 校验规则

以下错误会在预览页面标红：

1. **必填字段缺失**
   - SKU 编码/名称/数量为空
   - A 组和 B 组都为空

2. **格式错误**
   - 电话格式不正确
   - 数量不是正数
   - 温层不在允许范围内

3. **重复检测**
   - 同批次内外部编码重复
   - 与历史数据重复

### 修复错误

1. 直接在预览表格中点击错误单元格进行编辑
2. 错误修正后，标红会自动消失
3. 有错误的行不允许提交，需全部修正后再提交

## 性能优化

### 大文件处理（1000+ 条记录）

系统已自动优化：

- **虚拟列表**：只渲染可见区域的数据
- **分批解析**：避免阻塞 UI
- **内存管理**：自动释放已处理的数据

**预期性能**：
- 1000 条标准 Excel 数据：10 秒内完成
- 前端渲染：3 秒内完成


## 常见问题

### Q: AI 生成的规则不准确怎么办？

A: AI 生成的规则是推荐配置，您可以在保存前手动微调。建议：
1. 使用典型样例文件供 AI 分析
2. 仔细检查 AI 生成的字段映射
3. 复杂规则可先手动配置基础框架，再用 AI 补充

### Q: 如何批量修改已导入的运单？

A: 目前不支持批量修改。建议：
1. 导出需要修改的运单为 Excel
2. 在 Excel 中批量修改
3. 删除原记录，重新导入

### Q: 如何备份数据？

A: 数据已持久化到 PostgreSQL 数据库。可以：
1. 使用 Neon Dashboard 导出
2. 使用 Prisma 工具导出
3. 定期备份数据库

### Q: 支持图片识别吗？

A: 当前版本不支持图片 OCR。未来版本将支持：
- 图片中的表格解析
- 扫描件识别

### Q: 可以自定义字段吗？

A: 当前字段是固定的。自定义字段功能计划中。
